Organización

Estructura

Idealmente, las sesiones necesarias para llevar a cabo esta actividad son cinco:

  • Dos sesiones teóricas:
    • Una primera que explique todos los conceptos que se van a estudiar: Domótica, Internet of Things, automatización, Internet of Behaviors, Turismo sostenible, etc. Para ello, se dispone de una presentación.
    • La segunda sesión se usará para presentar todas las herramientas con las que se va a trabajar, dividir los grupos de trabajo y asignar el material correspondiente. También dispone de una presentación.
  • Dos sesiones prácticas:
    • La primera de ellas para programar sensores y actuadores que vaya a utilizar cada uno.
    • En la segunda sesión se buscará la realización de automatizaciones coherentes con los sensores, buscando la reducción de consumo. Si sobrara tiempo, se buscaría añadir Telegram para la realización de dichas automatizaciones.
  • Una última sesión donde cada grupo presente el trabajo realizado,  se realice un breve cuestionario de evaluación y un debate sobre las ideas vistas durante la práctica.

Hardware

El material utilizado para la implementación de esta actividad ha sido el siguiente:

  • Una Raspberry Pi 4 con 8GB de memoria RAM. Cualquier otro modelo podría servir. 
  • Un microcontrolador ESP32 para cada grupo de estudiantes. Modelos similares con conectividad WiFi también valdrían.
  • Sensores y actuadores compatibles con el microcontrolador elegido.
  • Cables y protoboards suficientes para las necesidades de los circuitos a realizar.

Software

Las herramientas software que se van a utilizar son las siguientes:

  • Home Assistant como base para crear el sistema domótico inalámbrico.
  • ESPHome, un complemento de Home Assistant que permite la configuración sencilla de las microcontroladoras.
  • NodeRED, otro complemento que facilita la conexión de Home Assistant con Telegram.
